# 🏕️ Генератор расписания лагеря «Прознание + Coddy»

Веб-приложение для построения расписания дневной смены детского клуба:
- собирает программу на 10 дней из библиотеки активностей
- учитывает выезды, ротацию отрядов, типы дней, конфликты педагогов
- публикует ссылку для педагогов (страница view.html — каждый видит свои занятия)
- генерирует PDF-тизер для родителей и подробную программу

**Где это работает:** [jane3dmp.github.io/camp-generator/](https://jane3dmp.github.io/camp-generator/)

# Часть 1 — Для пользователя

## Что вы получите

Открываете админку → за 5–30 минут собираете расписание смены → нажимаете «Опубликовать» → получаете ссылку которую отправляете педагогам. Каждый педагог открывает её, выбирает себя в выпадайке — видит свои занятия по дням.

## Главный workflow за 5 минут

1. **Заголовок смены**: тема (например «Зверополис»), стартовая дата, тип смены, возраст
2. **Тип каждого дня** — выбираете из пилюль над днём: «обычный», «выезд 3 утром», «выезд 6 после обеда» и т.п.
3. **Генерация активностей** — кнопка **⚡ сгенерировать всё** (внизу шапки) — заполняет все слоты случайным выбором из библиотеки
4. **Доводим вручную** — если что-то не нравится в конкретном слоте: клик по слоту → меню (заменить шаблон / закрепить ✓ / редактировать вручную)
5. **Публикуем** — кнопка **«Опубликовать»** в шапке → автоматически на сервер → ссылка готова

## Шапка админки — что для чего

| Кнопка | Что делает |
|---|---|
| **↶ отменить** | Отмена последней массовой операции (до 20 шагов). Хоткей **Ctrl+Z** |
| **⚡ сгенерировать всё** | Заполнить все незакреплённые слоты случайно из библиотеки |
| **🔄 все IT / Творч. / Активн.** | Перегенерация только одного направления. Учитывает кто уже занят |
| **🔓 снять закрепления** | Снять закрепления со всех слотов — после этого их можно перегенерить |
| **✏️ переименовать педагога** | Заменить имя педагога во всех слотах смены. Опция «во всей системе» — также в библиотеке Sheets и во всех публикациях (с бэкапом) |
| **🏷 переименовать шаблон** | То же самое для имени шаблона |
| **🧹 очистить темы** | Убрать поле «тема» из всех слотов (закреплённые не трогает) |
| **🪄 умная чистка рефов** | Убирает картинки которые не подходят к шаблону (рудименты от старой смены) |
| **🧹 очистить рефы** | Убрать **все** картинки из всех слотов (жёсткий вариант) |
| **🔍 проверить расписание** | Найти конфликты — один педагог в двух местах одновременно |
| **👤 имя** | Сменить имя редактора (видят другие если открыли ту же смену с другого компа) |

## Действия в карточке слота

В каждой карточке (после клика):
- **Заменить шаблон** — выбрать другую активность из библиотеки
- **Сгенерировать новый** — случайно из этого направления
- **✓ закрепить** — пометить «не трогать при перегенерации»
- **🔖 рефы** — выбрать какие картинки покажет педагог (по умолчанию все из библиотеки)
- **💬 коммент** — заметка для педагога («взять бусины 5мм», «без бумаги А3»)
- **✏️ редактировать вручную** — поменять педагога/кабинет/направление точечно

## Публикация и архив

**Кнопка «🌐 Опубликовать»** в шапке открывает панель:

- **Текущая ссылка** для педагогов (📋 копировать, ↗ открыть)
- **Все опубликованные смены** в архиве:
  - 📋 копировать ссылку, ↗ открыть как педагог
  - 📥 загрузить в админку (продолжить редактирование)
  - 🗑 удалить публикацию
- **🆕 Начать новую смену** — отвязывает текущий state от публикации, чтобы следующее «Опубликовать» создало новую запись

**Индикатор** возле кнопки публикации:
- Серая — публикаций нет
- Жёлтая «Опубликовать» — есть несохранённые правки
- Зелёная — только что опубликовано

## Совместная работа

Если ту же смену открыли в другой вкладке/на другом компе — модалка предложит:
- **👁 Только смотреть** — read-only, кнопки заблокированы
- **✏️ Редактировать всё равно** — последний кто опубликует, перезапишет правки

## PDF

- **📄 Тизер** — короткий PDF (2 страницы) для родителей в чат. Перед генерацией — модалка «Что включить в тизер»: галочками отмечаете нужные активности по направлениям. Выбор сохраняется со сменой
- **📑 Полная программа** — 6-страничный PDF с обложкой, расписанием, описаниями активностей по направлениям, событиями, ценами и контактами

## Если что-то пошло не так

- **Случайно нажала** «🧹 очистить рефы» / перегенерировала / переименовала → жмите **↶ отменить** или **Ctrl+Z**
- **Загрузила старую публикацию** через 📥, потерялся текущий state → жмите **↶ отменить** (вернёт state до загрузки)
- **Опубликовала со сломанным расписанием** → откройте архив, нажмите 📥 на предпоследней публикации → вернётесь, доработайте → **«🔄 Обновить публикацию»**

# Часть 2 — Для оператора-помощника

Вы поддерживаете библиотеку активностей в Google Sheets. От аккуратности заполнения зависит как работает генератор и что видят родители в PDF.

## Как открыть библиотеку

1. Откройте Google Sheets с библиотекой (ссылка от администратора)
2. Лист **«Активности»** — главный
3. Сверху появится меню **«Прознание»** — это команды от скрипта

## Лист «Активности» — структура

Каждая строка — один шаблон активности. Колонки:

| Колонка | Что там должно быть | Пример |
|---|---|---|
| **Статус** | `утверждено` или `черновик` | `утверждено` |
| **Шаблон занятия** | Название с маленькой буквы, без кавычек, кратко | `мк макраме` |
| **Примеры тем** | Темы через запятую, конкретные | `браслет, ловец снов, обвес для рюкзака` |
| **Педагог** | Имя как должно отображаться на view | `Афанасенко Женя` |
| **Направление** | Точно одно из существующих | `Творчество` |
| **Локация / кабинет** | Куда идёт педагог | `2 этаж` |
| **Возраст** | Целевая аудитория | `6–11 лет` |
| **Длительность (мин)** | Число | `60` |
| **Формат** | `МК` для мастер-класса, `уличная` для активности на улице | `МК` |
| **Описание** | Что именно ребёнок делает (для AI и педагога) | `Плетёт браслет из ниток мулине...` |
| **Pinterest / Google / YouTube** | Заполняется автоматически меню «Прознание» | — |
| **Референсы** | Конкретные URL картинок (по одной на строку) | — |

## ❌ Что НЕЛЬЗЯ делать в Sheets вручную

1. **❌ Менять имя педагога правкой ячейки.** В опубликованных сменах останется старое имя — у этого педагога перестанет показываться расписание.
   **Правильно**: открыть админку → **«✏️ переименовать педагога»** → галочка **«Во всей системе»** → новое имя. Скрипт обновит и Sheets, и все публикации, и сделает бэкап.

2. **❌ Менять имя шаблона правкой ячейки.** Та же проблема — в публикациях останется старое имя, рефы потеряются.
   **Правильно**: админка → **«🏷 переименовать шаблон»** → выбрать в дропдауне → новое имя.

3. **❌ Удалять строки шаблонов** которые могут быть в опубликованных сменах. У педагога просто не будет картинок и описания (он увидит имя — но без подробностей).

4. **❌ Менять заголовки колонок** в первой строке. Скрипт ищет по точным именам — если вы переименуете «Шаблон занятия» в «Название», всё сломается.

5. **❌ Удалять колонку «Статус».** Если её нет — все шаблоны внезапно становятся видны (включая черновики). Если случайно удалили — меню → **«Добавить колонку Статус»**.

6. **❌ Сложные форматы (формулы, объединённые ячейки)** в колонках с данными. Скрипт читает простой текст.

## ✅ Что МОЖНО и НУЖНО делать

1. **✅ Добавлять новые шаблоны** — просто новая строка в конец листа со статусом `утверждено`. Пишите всё в нижнем регистре кроме сокращений (МК, ИТ).

2. **✅ Редактировать описание** — описание копируется в state слота при первом создании, потом живёт отдельно. Изменение в библиотеке повлияет только на новые слоты, на старые — нет.

3. **✅ Редактировать длительность, направление, формат** для шаблонов которые **ещё не использовались** в публикациях. Если уже использовались — старые слоты в публикациях сохранят старые значения (что обычно ОК).

4. **✅ Расширять библиотеку через ИИ**:
   - Курсор на строку с шаблоном → меню **Прознание** → **🪄 Расширить выделенный шаблон**
   - ИИ предложит 4 похожих шаблона со статусом «черновик» (бежевый цвет)
   - Просмотрите, отредактируйте если нужно
   - Курсор на черновик → **✓ Утвердить** → попадает в HTML

5. **✅ Заполнять референсы** для всего листа: меню **Прознание** → **🔗 Заполнить референсы**. Создаёт колонки Pinterest/Google/YouTube с готовыми поисковыми запросами.

6. **✅ Добавлять конкретные картинки** в колонку «Референсы» — по одной URL на строку (Enter между ссылками). Эти картинки покажутся педагогу в его расписании.

## Лист «Реквизиты»

Простая таблица: ключ — значение. Используется в PDF (адрес, телефоны, цены, бренд).

Менять в Sheets можно свободно — это не привязано к публикациям.

## Лист «Типы смен»

Каталог типов лагеря (например «Тематический», «Городской», «Профильный»). Каждый со своим описанием для AI и якорными событиями. Менять можно свободно.

## Лист «Опубликованные версии»

❌ **РУКАМИ НЕ ТРОГАТЬ.** Это база всех опубликованных смен — каждая строка это снимок состояния расписания. Любая правка ячейки (особенно `stateJson`) сломает соответствующую публикацию.

Если нужно удалить публикацию — лучше через админку: 🌐 → 🗑.

## Лист «Сессии»

Технический лист — кто сейчас редактирует какую смену. Создаётся автоматически. Старые записи (старше 5 минут) удаляются скриптом сами. Не трогайте.

## Листы «Бэкап_yyyy-mm-dd_hh-mm-ss»

Создаются автоматически перед каждым каскадным переименованием педагога/шаблона. Копия листа «Опубликованные версии» на момент **до** изменения.

Можно удалять старые бэкапы (через месяц после переименования) — место освободится.

# Часть 3 — Для разработчика

## Стек

- **Frontend**: статические `index.html` + `view.html` на GitHub Pages, без билда. Весь JS внутри HTML
- **Backend**: Google Apps Script (Code.gs) развёрнут как Web App
- **БД**: Google Sheets (один документ, несколько листов)
- **AI**: Anthropic Claude (через прокси в Apps Script, ключ в Script Properties)

```
Браузер (admin/view)
    ↓ fetch
GitHub Pages (static HTML/JS)
    ↓ fetch с no-cors / GET
Apps Script Web App
    ↓ SpreadsheetApp / UrlFetchApp
Google Sheets / Anthropic API
```

## Файлы

- **index.html** (~12 000 строк, 76% — JS) — админка
- **view.html** (~1 200 строк) — страница педагога
- **Code.gs** (~1 800 строк) — серверная логика

## Локальная разработка

Никакого билда нет. Просто откройте `index.html` в браузере. Чтобы работало с реальными данными, нужно `LIBRARY_URL` в начале файла указывающий на ваш Apps Script.

```bash
git clone https://github.com/Jane3dMp/camp-generator.git
cd camp-generator
# редактируете в любом редакторе, открываете file:// в браузере
```

## Структура state (то что лежит в localStorage и в публикации)

```javascript
{
  theme: "Зверополис (Мир научных открытий)",
  startDate: "2026-06-15",
  campType: { type: "Тематический", description: "...", ... },
  ages: "6–11 лет",
  workWithoutThemes: false,
  publishId: "abc123def456",   // null если ещё не публиковали
  teaserSelection: ["мк макраме", "роблокс"],  // выбор для тизера
  schedule: [
    {
      type: "normal",  // normal | trip3_am | trip3_pm | trip3_full | trip6_am | trip6_pm | trip6_am_split | ...
      am_cluster: [ {slot}, {slot}, {slot} ],
      pm_cluster: [...],
      am_outside: [...],
      pm_outside: [...],
      // для выездов — отдельные поля trip6_am, trip_full_am, trip6_am_a/b и т.д.
    },
    // ... 10 дней
  ]
}
```

Структура **слота**:

```javascript
{
  template: "мк макраме",     // имя шаблона из LIBRARY (нижний регистр)
  name: "мк макраме",          // display name (может включать тему: "мк макраме «браслет»")
  theme: "",                   // конкретная тема, если задана
  teacher: "Афанасенко Женя",  // имя педагога
  room: "2 этаж",
  direction: "Творчество",
  format: "МК",
  duration: 60,
  locked: false,               // закреплено пользователем
  selectedRefs: [],            // [] = ничего; undefined = все из LIBRARY (фолбек)
  comment: "",                 // заметка для педагога
}
```

## Apps Script — actions

**GET** (`?action=...`):
- `library` / `all` — вся библиотека
- `campTypes`, `details`, `getRefs`, `previewUrl`, `imageProxy`
- `listPublished`, `getPublished` — публикации
- `listDrafts`, `loadDraft`
- `listSessions`

**POST** (JSON body с `action`):
- `claude` — прокси к Anthropic API (передаём prompt, system)
- `publish` / `unpublish`
- `saveDraft` / `deleteDraft`
- `addCustomTemplate`, `addRef`, `removeRef`, `appendThemeExample`
- `expandTemplate`, `approveTemplate`
- `heartbeat` / `endSession` — сессии
- `renameTeacher` / `renameTemplate` — каскадное переименование с бэкапом

## Критические места кода

1. **publishVersion_ в Code.gs** — `setValues()` обрезает строки длиннее 50000 символов при обновлении ячеек. Поэтому **нельзя** обновлять stateJson через setValues — нужно `deleteRow + appendRow`. Это касается и `_renameTeacherInPublications_` и `_renameTemplateInPublications_`.

2. **publishSmena в index.html** — POST идёт в `mode: 'no-cors'` (Apps Script не возвращает CORS-заголовки). Поэтому нет ответа — мы делаем verify-loop через GET getPublished, сравниваем fingerprint.

3. **getActiveRefsForSlot в view.html** — `selectedRefs === undefined` означает «использовать все из LIBRARY» (фолбек), `[]` означает «ничего не показывать» (явно очищено). Это специально согласовано с UI модалки 🔖 в админке.

4. **defaultSlot в index.html** — должна **всегда** возвращать `selectedRefs: []` и `comment: ''` (не undefined). Иначе при перегенерации могут «прилипнуть» старые рефы из других шаблонов.

5. **normalizeName** — приводит имя к канонической форме (lowercase + ё→е + лишние пробелы). Используется в view.html для поиска педагога и в Code.gs для каскадных переименований. **Не** покрывает реальные опечатки (О vs А) — для них есть кнопка «переименовать педагога».

## Как развернуть с нуля (новая копия)

### 1. Google Sheets
1. Создать новую Google Sheets
2. Создать листы: «Активности», «Типы смен», «Реквизиты»
   - Заголовки в «Активности» (первая строка): `Статус | Шаблон занятия | Примеры тем | Педагог | Направление | Локация / кабинет | Возраст | Длительность (мин) | Формат | Описание`
   - Заголовки в «Типы смен»: `Тип | Описание для AI | Стиль тем | Якоря (день:тип-дня:что) | Акценты направлений`
3. Заполнить хотя бы пару строк в «Активности» (со статусом `утверждено`)
4. Скопировать **SHEET_ID** из URL: `docs.google.com/spreadsheets/d/<<<SHEET_ID>>>/edit`

### 2. Apps Script
1. В Sheets → Расширения → Apps Script
2. В файле Code.gs **полностью** заменить содержимое на текущий Code.gs из репо
3. В строке `const SHEET_ID = '...'` подставить ваш ID
4. Project Settings → Script Properties → Add property:
   - `ANTHROPIC_API_KEY` = ваш ключ от console.anthropic.com (если используете AI)
5. Save (Ctrl+S)
6. Запустить функцию `_testLoadAll` — должны увидеть в Logs «Активностей: N»
7. **Deploy → New deployment → Web app**:
   - Execute as: **Me**
   - Who has access: **Anyone**
8. Скопировать **Web app URL**

### 3. GitHub Pages
1. Форкнуть/создать репо `camp-generator`
2. В файлах `index.html` и `view.html` найти строку `LIBRARY_URL = '...'` и подставить URL Apps Script
3. Settings → Pages → Source: `main` branch, root
4. URL вида `https://username.github.io/camp-generator/`

### 4. Тест
- Открыть `https://username.github.io/camp-generator/`
- Должна загрузиться админка, тост «Библиотека: N активностей»
- Если падает — F12 → Console, скорее всего CORS / неправильный LIBRARY_URL

## Обновление кода (после правок)

### HTML на GitHub Pages
1. Залить новый `index.html` или `view.html` через web-интерфейс GitHub
2. Подождать 30–60 сек на пересборку Pages
3. В браузере: `Ctrl+Shift+R` (полный сброс кэша) — иначе старая версия из CDN

### Apps Script
1. Save (Ctrl+S)
2. **Обязательно**: Deploy → Manage deployments → ✏️ Edit → Version: **New version** → Deploy
   - Если просто Save без New version — изменения не подхватываются клиентом
3. URL остаётся тот же — менять в HTML не нужно

## Известные ограничения

- **Apps Script лимиты**: 20 000 вызовов UrlFetchApp / день для бесплатного аккаунта. На реальной нагрузке (10 редакторов одновременно с heartbeat) хватает с запасом
- **Anthropic rate limits**: 30 000 input tokens / minute на тире 1. Расширение библиотеки на 8 шаблонов подряд может упереться → подождать 60 сек
- **CacheService Apps Script**: 100 КБ на один ключ. Изображения больше 100 КБ не кэшируются (но всё равно отдаются)
- **localStorage**: 5–10 МБ в браузере. State одной смены ~50 КБ → влезает 100+ копий
- **Файл index.html**: 12 000 строк → редактировать руками тяжело, но grep + str_replace справляются
- **Один HTML — один арендатор**: нет multi-tenant, бренд (Прознание+Coddy) и SHEET_ID зашиты в код

## Что **не** реализовано / возможные направления

- Аутентификация (сейчас админка доступна всем у кого есть URL)
- Multi-tenant (один Apps Script на несколько клубов)
- Rich-text в комментариях слотов
- Автоматическая публикация по расписанию
- Экспорт в Excel
- Мобильная версия админки (сейчас только на десктопе)
